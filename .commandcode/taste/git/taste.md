# git
- Review `git diff` and `git status` before creating commits. Confidence: 0.85
- Keep each commit focused on one logical change only. Confidence: 0.85
- Use Conventional Commits format: `<type>(optional-scope): <summary>`. Supported types: feat, fix, refactor, docs, test, chore, perf, build, ci. Confidence: 0.85
- Write commit summaries in imperative mood, lowercase, under 72 characters, with no period. Confidence: 0.85
- Add commit body only when useful, explaining "why", design decisions, or breaking changes. Confidence: 0.85
- Before committing, verify no debug code, commented-out code, temporary logs, secrets, or unrelated files. Confidence: 0.85
- If uncertain about a commit, summarize changes and propose a message for confirmation instead of committing immediately. Confidence: 0.85
- Never commit build artifacts, secrets, meaningless messages, or unrelated changes in one commit. Confidence: 0.85
